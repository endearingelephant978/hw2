RESPONSIVE REDESIGN

OVERVIEW

This project is a redesign of a website that has a few usability and accessibility problems. First I analyze these problems, then I make low-fidelity and high-fidelity prototypes for a redesign that addresses these problems, and finally I code a responsive page based on those prototypes.

shangrilatop100ma.com

The website I chose to analyze is the landing page for my favorite restaurant from my hometown, and while this is actually already a more recent redesign of their website (the older one left even more to be desired), there are still a few design problems to be addressed.

ORIGINAL DESIGN ANALYSIS

USABILITY PROBLEMS

PROBLEM EXAMPLE
superfluous links there are four separate links in the top half of the page that all take you to the online ordering page (grated because three of them are attached to the scrolling pictures, you only ever see at most two at once, but this is still more than is necessary)
misuse of spacing there is inconsistently a lot of space around every element; this isn't just an aesthetic issue, as the options in the navbar at the top of the page can be a bit hard to parse at first glance because the spacing is inconsistent
misuse of ordering the space for content at top of the page is not well utilized; most of the information there isn't helpful for new users (but instead shows up further down on the page), and the online ordering buttons should be even more prominent for experienced users to be able to beeline straight to them
misleading links there is a link on the page connected to a Google icon, and on the mobile site (but not on the desktop site for some reason) the text “Follow Us!” appears next to it, which implies some sort of Google social media site, but instead the link takes you to Google maps with the destination set to the address of the restaurant

ACCESSIBILITY PROBLEMS

PROBLEM EXAMPLE
excessive bandwidth three (out of a total seven) links that bring you back to this exact page actually unnecessarily reload the page, which causes an accessibility issue by using extra internet bandwidth when not required
text-on-image contrast the white text on the pictures overlaps with the food detail on the righthand side of the picture, making it hard to read (which only happens on mobile, so this actually could be addressed as a scaling issue)
missing alternative text according to the WAVE tool, there are some pictures on the page that are missing alternative text, which could cause trouble for people using screen readers

LO-FI WIREFRAMES

The following are lo-fi wireframe sketches for a redesign that address the issues I found in my analysis. There is a desktop wireframe, a tablet wireframe, and a mobile wireframe. Each design choice is annotated with number that corresponds to the usability/accessibility problem being addressed:

  1. superfluous links → collapsed all four different "order" buttons into one
  2. misuse of spacing → appropriately spaced separate elements in footer
  3. misuse of ordering → put key features of the website in a more natural order
  4. misleading links → reskinned the Google maps link to make its function clearer
  5. excessive bandwidth → removed links that unnecessarily reloaded the page
  6. text-on-image contrast → moved pictures out from behind any text
  7. missing alternative text → (intend to) include alternative text for all images
